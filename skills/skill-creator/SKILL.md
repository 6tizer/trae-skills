# 技能创建器 2.0

## 何时使用
- 用户要创建新技能
- 用户要改造已有技能
- 用户要评估技能触发准确性

## 面试式创建流程
1. 明确目标：技能要解决什么问题
2. 收集触发语：用户会说什么
3. 设计流程：触发后执行哪些步骤
4. 定义输出：交付格式是什么
5. 识别边界：失败、异常、特殊情况

## 写作原则
- 解释为什么，不只写必须做什么
- description 同时写“做什么 + 何时触发”
- 技能先可用，再迭代优化

## 技能结构
- `.trae/skills/<skill-name>/SKILL.md`（必需）
- `scripts/`（可选）
- `references/`（可选）
- `assets/`（可选）
